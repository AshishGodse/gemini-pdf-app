# Working Setup - No Docker Build Complexity

This guide provides a **guaranteed working setup** for local development without Docker build issues.

## Prerequisites

- **Windows 10+**, **macOS 10.15+**, or **Linux**
- **Docker Desktop** (for MongoDB and LocalStack only)
- **Node.js 18+** (https://nodejs.org/)
- **Python 3.11+** (https://www.python.org/)
- **npm** (comes with Node.js)
- **pip** (comes with Python)

## Step 1: Start Infrastructure Services (Docker containers)

### Windows (PowerShell)

```powershell
# Window 1: Start MongoDB
docker run -d `
  -p 27017:27017 `
  -e MONGO_INITDB_ROOT_USERNAME=root `
  -e MONGO_INITDB_ROOT_PASSWORD=root123 `
  --name pdf-mongo-dev `
  mongo:6.0

Write-Host "✓ MongoDB started on port 27017"
Write-Host "  Connection: mongodb://root:root123@localhost:27017/pdf_accessibility"
```

```powershell
# Window 2: Start LocalStack (S3 mock)
docker run -d `
  -p 4566:4566 `
  -e SERVICES=s3 `
  -v /var/run/docker.sock:/var/run/docker.sock `
  --name pdf-localstack-dev `
  localstack/localstack:latest

Write-Host "✓ LocalStack started on port 4566"
Write-Host "  S3 Endpoint: http://localhost:4566"
```

### macOS / Linux

## Stopping Services

### Stop Individual Services

```bash
# Gracefully stop services (keep running in terminals with Ctrl+C)
# Once stopped, containers still exist

# Remove containers
docker stop pdf-mongo-dev pdf-localstack-dev
docker rm pdf-mongo-dev pdf-localstack-dev
```

### Complete Cleanup

```bash
docker stop pdf-mongo-dev pdf-localstack-dev 2>/dev/null
docker rm pdf-mongo-dev pdf-localstack-dev 2>/dev/null
docker volume prune -f
```

## Troubleshooting

### Python Agent fails to start

**Error: `ModuleNotFoundError: No module named 'fastapi'`**

Solution:
```bash
cd python-agent
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```

**Error: `Port 8000 is already in use`**

Solution:
```bash
# Find and kill process on port 8000
# Windows:
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# macOS/Linux:
lsof -i :8000
kill -9 <PID>

# Or use different port:
python -m uvicorn app.main:app --reload --port 8001
```

### Backend fails to start

**Error: `npm ERR! code ERESOLVE`**

Solution:
```bash
cd backend
npm cache clean --force
rm package-lock.json
npm install --legacy-peer-deps
npm run dev
```

**Error: `Cannot find module 'typescript'`**

Solution:
```bash
cd backend
npm install typescript ts-node @types/node
npm run dev
```

### Frontend won't compile

**Error: `TypeScript errors`**

Solution:
```bash
cd frontend
npm cache clean --force
rm package-lock.json
npm install
npm run dev
```

**Error: `Cannot connect to backend`**

Check:
1. Backend is running: `curl http://localhost:5000/health`
2. API URL in `frontend/src/services/api.ts` matches your setup
3. Vite is set to use `http://localhost:5000/api` in dev mode

### MongoDB connection fails

**Error: `connect ECONNREFUSED`**

Solution:
```bash
# Verify MongoDB is running
docker ps | grep mongo

# Restart if needed
docker stop pdf-mongo-dev
docker rm pdf-mongo-dev
docker run -d \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=root \
  -e MONGO_INITDB_ROOT_PASSWORD=root123 \
  --name pdf-mongo-dev \
  mongo:6.0
```
